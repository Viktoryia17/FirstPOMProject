# **Автоматизация UI-тестирования веб\-приложения с использованием Playwright**

## **1\. Описание проекта**

Данный проект представляет собой систему автоматизированного тестирования веб\-приложения, реализованную с использованием фреймворка Playwright и языка программирования JavaScript.

Основной задачей проекта является проверка корректности работы пользовательских сценариев посредством end-to-end тестирования интерфейса приложения.

В рамках проекта автоматизированы основные действия пользователя:

– регистрация нового пользователя  
– авторизация в системе  
– проверка и обновление профиля пользователя  
– работа с корзиной товаров  
– оформление заказа  
– проверка обработки ошибок авторизации

Проект реализован с применением подхода Page Object Model, позволяющего разделить тестовую логику и взаимодействие со страницами приложения.

---

## **2\. Структура проекта**

Проект состоит из следующих основных компонентов.

### **Pages (Page Object Model)**

Директория содержит классы страниц приложения:

– LoginPage — работа со страницей авторизации  
– RegistrationPage — работа со страницей регистрации пользователя

В Page Object вынесены:

– селекторы элементов интерфейса  
– действия пользователя (ввод данных, нажатие кнопок)  
– методы получения текстов и сообщений об ошибках

### **Tests**

Содержит автоматизированные тестовые сценарии:

– тесты авторизации пользователя  
– регистрация и проверка данных профиля  
– обновление профиля пользователя  
– работа с корзиной  
– оформление заказа

---

## **3\. Используемые технологии**

В проекте используются следующие технологии:

– Playwright Test Runner  
– JavaScript (ES6)  
– Node.js  
– pnpm как менеджер пакетов  
– Page Object Model (POM)  
– переменные окружения (.env)

---

## **4\. Настройка окружения**

Для запуска тестов используется файл .env, содержащий данные пользователя:

PLAYWRIGHT\_USERNAME=user1@test.com  
PLAYWRIGHT\_USERPASSWORD=user123

Описание переменных:

– PLAYWRIGHT\_USERNAME — пользователь для авторизации  
– PLAYWRIGHT\_USERPASSWORD — пароль пользователя

Файл .env содержит конфиденциальные данные и не добавляется в репозиторий.

---

## **5\. Запуск тестов**

Запуск всех тестов выполняется командой:

pnpm playwright test

Запуск тестов с отображением браузера:

pnpm playwright test \--headed

В данном режиме браузер открывается визуально, что позволяет наблюдать выполнение тестов.

Запуск тестов в UI-режиме:

pnpm playwright test \--ui

UI-режим позволяет:

– запускать тесты по отдельности  
– отслеживать выполнение шагов  
– быстро находить ошибки  
– повторно запускать упавшие тесты

## **6\. Реализованные тестовые сценарии**

### **6.1 Авторизация пользователя**

Реализованы два сценария:

Успешная авторизация:

– открытие страницы логина  
– ввод корректных данных пользователя  
– выполнение входа в систему  
– проверка перехода на страницу каталога товаров

Негативный сценарий:

– ввод некорректных данных  
– проверка отображения сообщения «Неверный email или пароль»

---

### **6.2 Регистрация пользователя и проверка профиля**

Перед выполнением тестов используется механизм beforeEach, в котором выполняются следующие действия:

– регистрация нового пользователя  
– автоматический переход на страницу логина  
– авторизация созданного пользователя  
– проверка успешного входа в систему

Для создания уникальных пользователей применяется генерация данных с использованием Math.random.

После авторизации выполняется проверка страницы профиля:

– Email  
– Имя  
– Фамилия  
– Username  
– Телефон

Значения полей сравниваются с данными, введёнными при регистрации.

---

### **6.3 Обновление профиля пользователя**

Тест проверяет возможность изменения пользовательских данных.

Выполняются следующие действия:

– открытие страницы профиля  
– проверка текущих значений полей  
– изменение данных пользователя  
– сохранение изменений

Данный сценарий подтверждает корректную работу функциональности редактирования профиля.

---

### **6.4 Работа с корзиной**

После авторизации выполняются действия пользователя:

– добавление товара iPhone 15 Pro в корзину  
– переход в раздел «Корзина»  
– проверка отображения товара  
– удаление товара  
– проверка сообщения «Ваша корзина пуста»

---

### **6.5 Оформление заказа**

Реализован сценарий оформления заказа:

– добавление товара в корзину  
– переход к оформлению заказа  
– возврат пользователя на страницу каталога  
– проверка отображения сообщения «Заказ успешно создан\!»

---

## **7\. Используемые практики автоматизации**

В проекте применяются современные практики автоматизированного тестирования:

– использование Page Object Model для разделения логики тестов и страниц  
– применение beforeEach для переиспользования повторяющихся шагов  
– хранение конфиденциальных данных в переменных окружения  
– использование устойчивых локаторов Playwright (getByRole, getByLabel)  
– генерация уникальных тестовых данных  
– выполнение проверок с использованием expect
